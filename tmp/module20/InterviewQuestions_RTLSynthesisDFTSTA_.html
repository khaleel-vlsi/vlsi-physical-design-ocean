Q1.

You receive an RTL with no timing constraints yet. As a PD engineer, what RTL red flags do you look for immediately?

How to answer (Expected):

  • Deep combinational logic
  • Multiple always blocks driving same signal
  • Latches (incomplete assignments)
  • Clock gating written in RTL
  • Async resets on too many flops
  • Large fanout control signals

 Interviewer checks if you think beyond “not my job”

Answer:

  • Excessive muxing → large combinational cones
  • Wide enable logic replicated across blocks
  • Flat RTL without hierarchy → synthesis flattening
  • Overuse of case statements → logic duplication

 Mention routing congestion, not logic only

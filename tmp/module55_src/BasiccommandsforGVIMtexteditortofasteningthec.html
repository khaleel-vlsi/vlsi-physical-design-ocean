Modes of GVIM:

One of the most distinctive features of GVIM is its modal editing, where it operates in 4 different modes. They are,

  1. Escape mode / Normal mode
  2. Insert mode
  3. Visual block mode
  4. Recording mode
  5. Command line mode

  1. Escape Mode:

This is the default mode when you start Vim or switch back to it after editing text. In this mode, you can navigate, delete, copy, paste, and perform various other operations using key combinations. Pressing the "Escape" key in Normal Mode allows you to ensure you're not in any other mode and are ready to execute commands.

  1. Insert Mode:

This is the mode where you can actually type and edit text. To enter Insert Mode from Normal Mode, you can press "i" (for insert) or "a" (for append). While in Insert Mode, you can type and edit as you would in any other text editor.

  1. Visual Block mode:

Visual block mode in Vim (and GVim) allows you to select and edit a rectangular block of text rather than entire lines. This can be very useful for making column-based changes in your text.

We can enter by pressing “Ctrl+v” from normal mode.

  1. Recording Mode:

In Vim, you can use the recording feature to record a sequence of actions as a macro and then replay that macro on different parts of your text. This can be incredibly useful for automating repetitive tasks.

We can go through recording mode by pressing q addition with any alphabet i.e qa, qd, qf ,. from normal mode and record what we type and the exit by pressing q

  1. Command line Mode:

This mode is used for entering more advanced commands, such as searching, replacing, saving files, and so on. To enter Command-Line Mode from Normal Mode, press ":". From here, you can type various commands and press "Enter" to execute them.

VIMRC File:

In Vim, including the GUI version like GVim, the vimrc file is a configuration file that contains various settings and customizations for the Vim text editor. The name "vimrc" stands for "Vim run commands," and it can provide additional options related to its graphical interface, like window placement, color schemes, and menu configurations.

You can create and edit your vimrc file to modify a wide range of Vim's behaviors and features.

To create or edit your vimrc file for GVim, you generally need to locate or create the file in your user's home directory with the name .vimrc. On some systems, it might be called _vimrc. The underscore version is used on Windows systems.

Remember that changes you make in the vimrc file take effect the next time you start Vim or GVim. Some common customizations that you might put in your vimrc include:

  1. Basic Settings: You can set things like line numbers, tab behavior, and word wrap settings.
  2. Key Bindings: You can define custom key mappings or modify existing ones to make navigation and editing more efficient for you.
  3. Plugins and Extensions: Vim supports various plugins that enhance its capabilities. You can use your vimrc to manage these plugins and their settings.
  4. Color Schemes: You can set the color scheme for the interface, which affects the appearance of the text and the user interface elements.
  5. Autocommands: These are commands that Vim automatically executes in response to certain events, like opening a file or saving a buffer. You can define autocommands in your vimrc.
  6. Search and Replace Options: Configure how search and replace operations behave, like case sensitivity and highlighting.
  7. Filetype-specific Settings: You can configure different settings for specific file types. For example, you might want different settings for programming files compared to Markdown files.

Common commands for VIMRC file

set number        Set line numbers

set guifont:fixedsys:h14        Set the font size and color

set showmode        Show the current mode in the status line.

set tabstop=4        Set the tabspace with 4units.

set nocompatible        Disable vi-Compatible Mode.

set backspace=indent,eol,start        Customize Backspace behavior.

  1. indent: Allows the Backspace key to delete automatic indentations added by Vim when you press Enter. This is useful for maintaining consistent indentation levels.
  2. eol: Allows the Backspace key to delete the newline character at the end of a line, effectively merging the current line with the previous line.
  3. start: Allows the Backspace key to delete the leading whitespace at the start of a line, which can be useful for quickly removing indentation.
